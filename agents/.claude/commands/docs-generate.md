# Docs Generate - Generate Documentation

Generate documentation for code, APIs, or features.

## When to Use

- New feature needs documentation
- API endpoints need docs
- Complex code needs explanation
- README needs updating

## Documentation Types

### Code Documentation
- JSDoc/TSDoc comments
- Function descriptions
- Parameter documentation
- Return type documentation

### API Documentation
- Endpoint descriptions
- Request/response examples
- Authentication requirements
- Error responses

### Feature Documentation
- User-facing guides
- How-to instructions
- Configuration options
- Troubleshooting

### Architecture Documentation
- System overview
- Component relationships
- Data flow diagrams
- Decision records

## Generation Steps

### Step 1: Identify Scope

What needs documentation?
- Specific file/function
- API endpoint
- Feature/module
- Entire system

### Step 2: Read the Code

Understand what the code does before documenting:
- Read implementation
- Trace data flow
- Identify edge cases

### Step 3: Find Examples

Look for existing documentation patterns in the project.

### Step 4: Generate Docs

Create documentation following project conventions:
- Match existing style
- Use appropriate format (markdown, JSDoc, etc.)
- Include examples where helpful

### Step 5: Review

Verify documentation is:
- Accurate
- Complete
- Clear
- Consistent with codebase

## Documentation Best Practices

DO:
- Document the why, not just what
- Include examples
- Keep docs close to code
- Update docs with code changes

DONT:
- Document obvious code
- Let docs get stale
- Write novels (be concise)
- Duplicate information
