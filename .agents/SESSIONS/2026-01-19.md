### Skill Details

The humanizer skill identifies and removes 24 AI writing patterns:

**Content Patterns:**

- Undue emphasis on significance/legacy
- Undue emphasis on notability/media coverage
- Superficial -ing analyses
- Promotional language
- Vague attributions/weasel words
- Outline-like "Challenges and Future Prospects" sections

**Language Patterns:**

- Overused AI vocabulary (delve, crucial, pivotal, landscape, etc.)
- Copula avoidance (serves as/stands as instead of is/are)
- Negative parallelisms (It's not just X, it's Y)
- Rule of three overuse
- Elegant variation (synonym cycling)
- False ranges

**Style Patterns:**

- Em dash overuse
- Overuse of boldface
- Inline-header vertical lists
- Title case in headings
- Emojis
- Curly quotation marks

**Communication Patterns:**

- Collaborative artifacts (I hope this helps!)
- Knowledge-cutoff disclaimers
- Sycophantic tone

**Filler/Hedging:**

- Filler phrases
- Excessive hedging
- Generic positive conclusions

Plus "soul injection" guidance for adding personality to clean-but-sterile writing.

### Source

Based on [Wikipedia:Signs of AI writing](https://en.wikipedia.org/wiki/Wikipedia:Signs_of_AI_writing), maintained by WikiProject AI Cleanup.

## Session 2: Claude Code Autocomplete Investigation

**Focus:** Investigating why `/clean` gets highlighted before `/clear` in CLI autocomplete

### Investigation Summary

User reported that when typing `/clear` in Claude Code CLI, the autocomplete dropdown highlights `/clean` instead of the exact match.

### Findings

1. **Root Cause Identified:**
   - User has global commands in `~/.claude/commands/` including `clean.md` and `end.md`
   - When typing `/clear`, both `/clear` (exact match) and `/clean` (prefix match) compete
   - Alphabetically `/clean` < `/clear`, causing wrong selection

2. **Recent Fix Exists:**
   - [Commit 870624f](https://github.com/anthropics/claude-code/commit/870624fc1581a70590e382f263e2972b3f1e56f5) (~2 weeks ago)
   - Fix: "prioritizing exact and prefix matches on command names over fuzzy matches"

3. **User's Version:** 2.1.12

### Recommendations Given

- Run `claude update` to ensure latest version
- If still occurring after update, open issue at https://github.com/anthropics/claude-code/issues

### Files Changed

None - investigation only

### Related Issues

- [Issue #9768](https://github.com/anthropics/claude-code/issues/9768) - Tab complete selecting wrong command

### Why agent-browser > webapp-testing

| webapp-testing | agent-browser |
|----------------|---------------|
| Python scripts | Direct CLI commands |
| CSS selectors | AI-optimized `@e1`, `@e2` refs |
| Complex setup | Just `npm install -g agent-browser` |
| Manual element finding | `snapshot -i` returns refs automatically |

### Key Commands

```bash
agent-browser open <url>        # Navigate
agent-browser snapshot -i       # Get interactive elements with @refs
agent-browser click @e1         # Click by ref
agent-browser fill @e2 "text"   # Fill input
agent-browser screenshot path.png
agent-browser --session qa1 open site.com  # Parallel sessions
```
