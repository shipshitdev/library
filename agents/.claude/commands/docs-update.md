# Docs Update - Update Documentation

Update existing documentation to reflect code changes.

## When to Use

- After code changes
- When docs are outdated
- After feature modifications
- During refactoring

## Update Steps

### Step 1: Identify What Changed

Review recent changes:
- Git diff
- Modified files
- New features
- Removed functionality

### Step 2: Find Related Docs

Locate documentation that needs updating:
- README files
- API docs
- Code comments
- Architecture docs
- Session files

### Step 3: Update Docs

Make necessary updates:
- Reflect new behavior
- Remove outdated info
- Add new sections if needed
- Update examples

### Step 4: Verify Consistency

Ensure all related docs are consistent:
- No contradictions
- Terminology matches
- Examples work

## Documentation Locations

Common documentation locations:
- README.md - Project overview
- docs/ - Detailed documentation
- .agent/SYSTEM/ - Architecture docs
- .agent/SESSIONS/ - Session history
- Code comments - Inline documentation

## Update Checklist

After code changes, check:
- README still accurate?
- API docs reflect changes?
- Architecture docs current?
- Code comments updated?
- Examples still work?

## Best Practices

- Update docs with code (same commit)
- Keep docs minimal but complete
- Remove outdated information
- Verify examples still work
